Virtual Machines House app

Virtual Machines House app is a light weight app designed to create and manage Mac and Linux virtual machines on mac computers.

Mac virtual machines

At present, mac virtual machines can be created only on the Apple Silicon based mac computers. Only macOS Monterey and above versions based virtual machines can be created.

Linux virtual machines

To create a virtual Linux system on Apple Silicon, an arm64 based ISO image file that supports virtiofs is required.
For example to create Alpine Linux virtual machine, download aarch64 iso file under VIRTUAL section at https://alpinelinux.org/downloads/ and use it in the app.

To create a virtual Linux system on Intel (x86_64) processor based systems, a x86_64 based ISO image file that supports virtiofs is required.
For example to create Alpine Linux virtual machine, download x86_64 iso file under VIRTUAL section at https://alpinelinux.org/downloads/ and use it in the app.

Shared directories on Mac virtual machines

Shared directories in macOS VMs are only available in macOS 13 and later.
The shared directories are automatically mounted at /Volumes/My Shared Files/. They can be seen in the Finder.

Shared directories on Linux virtual machines with virtio file system

To mount a shared directory of the host machine onto a directory of the guest machine, execute the following command on a terminal in the virtual machine.
mount -t virtiofs shared-folders directory
For example, to mount the shared directories of the host machine on to the /tmp folder of the guest virtual machine, execute:
mount -t virtiofs shared-folders /tmp

macOS Restore Image URI of IPSW files

Mac images links table has 3 sets of the URIs of IPSW files:

  1. Hardcoded set
  2. Populated set
  3. User-Given set

Hardcoded set

This set contains the URIs that are built-in with the app. Those are URIs of:

  1. macOS Monterey 12.6.1
  2. macOS Ventura 13.6
  3. Sonoma 14.6.1
  4. Sequoia 15.6.1

Populated set

App periodically checks for the latest macOS version. The resulting URI is saved in this set.

User-Given set

User provided URIs are in this set.

Terminology

Apple Silicon

Mac computers with ARM64 processor or M1/M2/M3/M4 chip. You can find this in "About this Mac" window. To open "About This Mac" click the Apple menu (ï£¿) in the top-left corner of your screen and select About This Mac.

Mac computers

A computer made by Apple that has an operating system called macOS. Example: MacBook Pro, MacBook Air, Macmini, Mac Studio, and iMac.

Host machine

A Mac machine that runs Virtual Machines House app.

Virtual machine

A guest macOS or Linux based systems running with the Virtual Machines House app.

ASIF - Apple Sparse Image Format

A modern sparse read/write image, introduced by Apple on macOS Tahoe. The space it uses on your disk is proportional to the data it contains, making it efficient and general-purpose.

Observation: It starts with just 4 MB and expands as and when needed.

Known limitations

In-built Camera cannot be accessed. Some Linux virtual machines tend to be inresponsive sometimes when shutting down or when left idle. Some system resources/ports may not be accessible to virtual machines. Performance of the virtual machines can be far less that of the real hardware machines.

On mac computers with Intel processor, the app cannot save and restore the state of the virtual machines.

Has following key changes:

  1. Introduced ASIF

  3. Introduced Screenshot Menu
  4. Minimum OS - This was set to macOS 15.6 from 15.2.
  5. Removed Latest Mac Image info fetch timer
  6. Improved Remove State button

ASIF

Disk images are created with ASIF format by default. Please note that this will allow creation of several virtual machines as the immediate size of the disk created is just 4 MB. So, if the warning about the disk size for a new virtual machine is not shown, it does not mean that there is sufficient space left. The disk of the existing virtual machines can grow per use anytime.

Screenshot Menu

A menu item is added to open app's screenshots folder in Finder. This will show up the app's container folders. Please take care not to disturb the containers folders and files.

Screenshot Button

It is on virtual machine windows to take screenshot of the respective window. It will be visible only when the virtual machine is on.

Latest Mac Image info fetch timer

Applicable only for Apple Silicon based mac systems. In previous versions it was fired once at a launch time and later regularly at a random interval between a few hours and a day. This created a clutter of image informations for some people who never needed. (Thinking to re-introduce the same timer disabled by default with a setting to enable for those who need it in a future version).

Improvements on Remove state button

This button was previously shown only on error. Now, it is shown whenever the saved state is detected. Also, The button's icon was previously Trash. It could confuse with the Remove virtual machine button. Therefore, it is now with a new icon - back arrow with cross mark, indicating the removal of state saved sometime back. Also, added a confirmation dialog before removing the state when the user clicks on the button. Also, added the button to the toolbar of the vm window.

Focus improvements

In the create VM window, focus was not on the name field when the field appeared. In this version, this is fixed.

Known issues

macOS Tahoe support on macOS Sequoia

This Device Support Image has to be installed beforehand. Otherwise, the following error will be observed:

Installing restore image failed. Error: A software update is required to complete the installation. Installation requires a software update.

Closed issues with version 1.3

Linux virtual machine restore issue

On Apple Silion, if a linux virtual machine is running when the app is quitting, the state of the virtual machine is stored but it becomes not usuable when the app is launched again. The state has to removed with the Remove State button or by other means manually. This is the workaround. This issue is fixed in 1.3.

Running state when restoring

On Apple Silicon, when restoring the virtual machine from saved state, the virtual machine goes to Paused state. User has to click on Resume to run the virtual machine. In 1.3, auto-resume is done. So, one step is reduced. Further, on resuming, just a "Running" state was shown in 1.2. This is fixed in 1.3. Virtual machine console window is shown now.

Closed issues with version 1.2

CPU count

Virtual Machines House app version 1.2 computes CPU count to half the size of the system's and uses the max of that with value 4. This solves the issue seen in version 1.1. Sample steps are captured at this page.

Virtual Machines House app version 1.1 by default used CPU count value 20. This caused some troubles. Edit the slide and set to 4. Troubleshooting with CPU value 20 is captured at this page.
