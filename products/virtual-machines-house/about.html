Virtual Machines House (VMH) Application

Virtual Machines House app is a light weight app designed to create and manage Mac and Linux virtual machines on mac computers.

Per study, on the AppStore, this is the :

  • First app to show the list of virtual machines in a table view facilitating sorting on various aspects.
  • First app to show IP address of the virtual machine.(Feature introduced in version 1.5.0).

Mac virtual machines

At present, mac virtual machines can be created only on the Apple Silicon based mac computers. Only macOS Monterey and above versions based virtual machines can be created.

Linux virtual machines

To create a virtual Linux system on Apple Silicon, an arm64 based ISO image file that supports virtiofs is required.
For example to create Alpine Linux virtual machine, download aarch64 iso file under VIRTUAL section at https://alpinelinux.org/downloads/ and use it in the app.

To create a virtual Linux system on Intel (x86_64) processor based systems, a x86_64 based ISO image file that supports virtiofs is required.
For example to create Alpine Linux virtual machine, download x86_64 iso file under VIRTUAL section at https://alpinelinux.org/downloads/ and use it in the app.

Shared directories on Mac virtual machines

Shared directories in macOS VMs are only available in macOS 13 and later.
The shared directories are automatically mounted at /Volumes/My Shared Files/. They can be seen in the Finder.

Shared directories on Linux virtual machines with virtio file system

To mount a shared directory of the host machine onto a directory of the guest machine, execute the following command on a terminal in the virtual machine.
mount -t virtiofs shared-folders directory
For example, to mount the shared directories of the host machine on to the /tmp folder of the guest virtual machine, execute:
mount -t virtiofs shared-folders /tmp

macOS Restore Image URI of IPSW files

Mac images links table has 3 sets of the URIs of IPSW files:

  1. Hardcoded set
  2. Populated set
  3. User-Given set

Hardcoded set

This set contains the URIs that are built-in with the app. Those are URIs of:

  1. macOS Monterey 12.6.1
  2. macOS Ventura 13.6
  3. Sonoma 14.6.1
  4. Sequoia 15.6.1

Populated set

When a user checks for the latest macOS version, the resulting URI is saved in this set. Similarly, when a user checks for a macOS developer beta, the resulting URI is saved in this set.

User-Given set

User provided URIs are in this set.

Terminology

Apple Silicon

Mac computers with ARM64 processor or M1/M2/M3/M4 chip. You can find this in "About this Mac" window. To open "About This Mac" click the Apple menu () in the top-left corner of your screen and select About This Mac.

Mac computers

A computer made by Apple that has an operating system called macOS. Example: MacBook Pro, MacBook Air, Macmini, Mac Studio, and iMac.

Host machine

A Mac machine that runs Virtual Machines House app.

Virtual machine

A guest macOS or Linux based systems running with the Virtual Machines House app.

ASIF - Apple Sparse Image Format

A modern sparse read/write image, introduced by Apple on macOS Tahoe. The space it uses on your disk is proportional to the data it contains, making it efficient and general-purpose.

Disk images are created with ASIF format by default. Please note that this will allow creation of several virtual machines as the immediate size of the disk created is just 4 MB. So, if the warning about the disk size for a new virtual machine is not shown, it does not mean that there is sufficient space left. The disk of the existing virtual machines can grow per use anytime.

Observation: It starts with just 4 MB and expands as and when needed.

NAT - Network Address Translation network mode

A NAT network mode isolates virtual machines. That means, only host can access the virtual machine by using the virtual machine's IP address. Another virtual machine or another host on the LAN cannot access the virtual machine.

This is the default mode used at the time of virtual machine creation. This can be changed later at the inspector pane on the main window or the respective virtual machine's window.

Bridge network mode

A Bridge network mode bridges virtual machines. That means, another virtual machine or another host on the LAN can access the virtual machine by its IP address.

When the mode is just "Bridge", app chooses the first network interface it 'sees' to bridge with.

Bridge with interface network mode

The difference from the previous is that app uses the given interface to bridge with.

Bookmarks

Following are bookmarked:

  • Folders given to mount on the virtual machines.
  • macOS restore images (IPSW files) given to create a macOS virtual machine.
  • Linux images (ISO files) given to create a Linux virtual machine.
Bookmarks can be viewed and managed at Bookmarks window. Click on the Window menu and click on Bookmarks submenu to open the Bookmarks window.

Screenshot Menu

A menu item opens app's screenshots folder in Finder. This will show up the app's container folders. Please take care not to disturb the containers folders and files.

Screenshot Button

It is on virtual machine windows to take screenshot of the respective window. It will be visible only when the virtual machine is on.

Latest Mac Image

On macOS Sequoia, Latest images URI points to just latest version of Sequoia and it does not point to macOS Tahoe.

Known limitations

ASIF not supported on macOS 26.3

Due to a technical change in macOS 26.3, disk creation gets stuck with Apple Sparse Image Format (ASIF). Please use Raw format when creating a virtual machine or when adding a disk to an existing virtual machine.

In-built Camera not supported

In-built Camera cannot be accessed. Some Linux virtual machines tend to be inresponsive sometimes when shutting down or when left idle. Some system resources/ports may not be accessible to virtual machines. Performance of the virtual machines can be far less that of the real hardware machines.

Intel - macOS virtual machines not supported

On mac computers with Intel processor, the app cannot be used to create a macOS virtual machine.

Intel - Save and Restore not supported

On mac computers with Intel processor, the app cannot save and restore the state of the virtual machines.

The app cannot be used on a virtual machine.

Apple Silicon - Media & Purchases not supported

On macOS virtual machines, Media & Purchases is not supported. AppStore may not accept sign-in.

Apple Silicon - iCloud support only - macOS 15 and above

iCloud support for virtual machines is available on macOS 15 and later only.

Reference: iCloud on virtual machine

Apple Silicon - macOS Tahoe support on macOS Sequoia

This Device Support Image has to be installed beforehand. Otherwise, the following error will be observed:

Installing restore image failed. Error: A software update is required to complete the installation. Installation requires a software update.

Apple notes

iCloud

Virtualization configures an identity for the VM that it derives from security information in the host’s Secure Enclave. Just as individual physical devices have distinct identities based on their Secure Enclaves, this identity is distinct from other VMs.

If someone moves a VM to a different Mac host and restarts it, the Virtualization framework automatically creates a new identity for the VM using the information from the Secure Enclave of the new Mac host. This identity change requires the person using the VM to reauthenticate to allow iCloud to restart syncing data to the VM.

Additionally, the Virtualization framework detects attempts to start multiple copies of the same VM simultaneously on the same Mac host. For example, when someone duplicates the files that make up a VM, the framework treats the copy of the VM as a clone of the first one. Starting a second clone while another clone is already running causes the Virtualization framework to automatically construct a new identity for the second clone. This preserves the integrity that different VMs have distinct identities, and requires that the person using the VM reauthenticate to use iCloud services.

Please consult iCloud support resources
